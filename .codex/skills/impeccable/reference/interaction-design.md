# Interaction Design

- Design default, hover, focus, active, disabled, loading, error, and success states.
- Never remove focus without a visible replacement.
- Use visible labels, not placeholders as labels.
- Validate on blur, not every keystroke, unless there is a good reason.
- Prefer skeletons over generic spinners when content shape matters.
- Prefer undo over confirmation for reversible actions.
- Use accessible primitives like dialog, popover, inert, and anchor positioning when appropriate.
